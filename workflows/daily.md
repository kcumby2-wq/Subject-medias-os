# Daily Workflow — Run Every Session In Order

## STEP 1 — FEEDBACK CHECK (always first)
Open `feedback/feedback-log.md`. New entries?
Process each:
- a. Identify which file it affects
- b. Propose the edit to Kyron
- c. On approval: make the edit, log in `feedback/improvements.md`

No new feedback? Move on.

## STEP 2 — OUTREACH PIPELINE CHECK
Open `data/outreach/outreach-log.md`
- Any program or tournament org that opened Email 2 but hasn't responded by Day 10? → add to `data/outreach/call-list.md`
- Any agreement sent but not signed after 5 days? → flag for follow-up
- Any events booked this week? → create event file in `data/events/`

## STEP 3 — EVENT PIPELINE CHECK
Scan `data/events/` for any event within 7 days:
- Pre-event logistics email sent? If not → draft and flag for send
- Creator crew assigned? If not → flag immediately
- QR code and consent forms ready? If not → flag immediately
- Post-event: footage logged? Assignments sent? Families notified?

## STEP 4 — GOALS PULSE
Open `goals/goals.md`. One line per goal: did anything move this forward? anything blocked? Output a 5-line status pulse at the top of the session.

## STEP 5 — TODAY'S PRODUCTION
Based on goals and pipeline status, produce today's work:
- **Outreach day** → write sport-specific email variations → `output/sequences/`
- **Content day** → write social posts in SM voice → `output/content/`
- **Event day** → produce event brief and creator assignments → `output/events/`
- **Report day** → weekly or monthly report → `output/reports/`

## STEP 6 — CLOSE THE LOOP
- Every deliverable saved to `output/` with date in filename
- Session log appended to `output/reports/session-log.md`
- List anything waiting on Kyron before ending

---

**Rule:** if a session ends and nothing new exists in `output/` the session failed. Files are the proof of work.
