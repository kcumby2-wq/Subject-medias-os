# Weekly Workflow — Monday Planning + Friday Review

## Monday Planning

1. Run the daily workflow (`workflows/daily.md`) first.
2. Review the week ahead: any booked events (`data/events/`), Pylon outreach sequence milestones due (Day 10/Day 5 follow-up triggers per `data/outreach/outreach-log.md`), creator assignments needed for upcoming shoots.
3. Decide the production focus for each day of the week — outreach day / content day / event-prep day / creator-pipeline day / report day.
4. Note any blockers that need Kyron's input before the week starts (agreement approvals, creator onboarding decisions, pricing exceptions).

## Friday Review

1. Run the daily workflow first.
2. Pull the week's numbers against `goals/goals.md`:
   - Individual Athlete packages sold (Goal 1) and package mix
   - Club Programs moved through the GHL pipeline stages, and any new bookings (Goal 2)
   - Tournament Organizations contacted/booked (Goal 3)
   - Pylon contacts moved past "Contacted" this week (Goal 4) — pull straight from `data/outreach/outreach-log.md`
   - Creators onboarded or advanced a level (Goal 5)
   - Deliveries completed, SR upsells sent, SR upsells converted (Goal 6)
3. Write the weekly report and save to `output/reports/`.
4. Flag anything that should roll into next Monday's planning.

## Self-Improvement Loop (run after the Friday Review numbers are in)

Per `SUBJECT-OS-MASTER.md` (`Groundfloorsports/trail-of-joy/`), this is the same six-step loop every Subject brand runs weekly:

1. **Review** — the six goals above, plus anything notable from `feedback/feedback-log.md` this week.
2. **Identify Bottlenecks** — e.g. outreach stalling at "Contacted" with no responses, creators onboarded but never assigned, deliveries happening but SR upsell conversion flat.
3. **Recommend Solutions** — e.g. a new outreach SOP, a creator-assignment checklist, a tighter SR upsell follow-up sequence.
4. **Prioritize** — rank by Highest Impact, Lowest Effort, Fastest Revenue.
5. **Update System** — edit the relevant `workflows/`, `skills/`, or `data/` file this same session, don't just note it for later.
6. **Archive Learning** — log the win, failure, or pattern in `feedback/improvements.md`, and in `LEARNING-LOOP.md` (`Groundfloorsports/trail-of-joy/`) if it's a cross-brand learning, not just a Subject Media one.

## Notes

Treat the first few Friday Reviews as calibration: the targets in `goals/goals.md` are real but unproven against actual weekly output yet. Adjust them honestly rather than padding the numbers to hit them.

The first Friday Review this runs against will likely show zeroes in several places — that's expected and the actual point: this workflow is what turns "the pipeline is set up in CLAUDE.md" into "the pipeline has real entries."
